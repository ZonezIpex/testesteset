#### 배경
사내에서 사용하는 IT 자산(노트북, 모니터, 태블릿, 모바일 단말 등)은 프로젝트 투입·부서 이동·외주/협력사 투입 같은 상황에서 짧은 주기로 배정과 회수가 반복됩니다. 하지만 실제 운영에서는 대여/반납 기록이 메신저나 엑셀, 담당자 개인 기록으로 분산되는 경우가 많아 “누가 어떤 자산을 언제부터 사용 중인지”, “반납 시 검수 결과가 무엇인지”, “파손·연체 비용을 어떤 근거로 산정했는지”가 한 번에 추적되지 않는 문제가 발생합니다.

이 과정에서 자산 분실/중복 지급/재고 불일치 같은 운영 리스크가 커지고, 반납 검수 기준이 사람마다 달라 파손·수리비 정산에서 분쟁이 생기기도 합니다. 특히 보증금 환급·차감과 연체료 산정이 계약/검수 이력과 분리되면 정산 근거가 약해져 관리 난이도가 급격히 올라갑니다.

본 프로젝트는 조달(PO→배송→입고→검수)부터 자산화, 대여/반납, 파손·연체 정산까지의 흐름을 데이터 모델로 고정해, 상태 역전(입고 전 대여 등)과 근거 누락(검수 없이 정산 등)을 예방하고 추적 가능한 자산 관리 체계를 설계하는 것을 목표로 합니다.

> (근거자료/캡처를 붙일 자리)
> - 사내 운영 사례(익명화): 엑셀 대장 캡처, 반납 누락 사례, 파손 정산 분쟁 사례 등
> - 참고 기사/자료: 링크 + 캡처 이미지
